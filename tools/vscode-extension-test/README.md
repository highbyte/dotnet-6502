# Folder Structure Explanation

This repo has two separate folders for the debugger:

## 1. `tools/vscode-extension/` - Extension Source Code

**Purpose**: Developing the VSCode extension itself

**Contents**:
- Extension source code (TypeScript)
- `.vscode/launch.json` - Launches **Extension Development Host**
- `.vscode/tasks.json` - Compiles the extension
- Test program source for reference

**How to use**:
1. Open this folder in VSCode
2. Press F5 → Extension Development Host window opens
3. This is the "extension development" window

## 2. `tools/vscode-extension-test/` - Test Workspace

**Purpose**: Testing/using the 6502 debugger

**Contents**:

Simple ca65 assembler example
- `samples/test-program.prg` - 6502 program to debug
- `samples/test-program.asm` - Source code (ca65 assembler)
- `samples/test-program.dbg` - Source debug info (generated by ca65 assembler)
- Optional: `samples/build-test-program.ps1` - Build script for manual build - not required if using built-in ca65 build task in .vscode/tasks.json
- Optional: `samples/build-test-program.sh1` - Build script for manual build -not required if using built-in ca65 build task in .vscode/tasks.json

Simple ACME assembler example
- `samples/test-program2.prg` - 6502 program to debug
- `samples/test-program2.asm` - Source code (ca65 assembler)
- `samples/build-test-program2.ps1` - Build script for manual build - required because no built-in ACME assembler build task
- `samples/build-test-program2.sh1` - Build script for manual build - required because no built-in ACME assembler build task

Full ca65 assembler C64 example (build with build task)
- `samples/smooth_scroller_and_raster2.prg` - 6502 program to debug
- `samples/smooth_scroller_and_raster2.asm` - Source code (ca65 assembler)
- `samples/smooth_scroller_and_raster2.dbg` - Source debug info (generated by ca65 assembler)

- `.vscode/launch.json` - Debugs the different example **6502 programs** (not the extension) with different configuration

**How to use**:
1. Open this folder in the **Extension Development Host** window
2. Press F5 → Debugs test-program.prg with the 6502 debugger
3. This is the "using the debugger" window

## Why Two Folders?

- **Separation of concerns**: Extension code vs. programs to debug
- **Avoid confusion**: Each folder has its own `.vscode/launch.json` for different purposes
- **Clean testing**: Test workspace can have multiple .prg files without cluttering extension source

## For End Users

When the extension is published, users only need:
- The `tools/vscode-extension-test/` pattern (any folder with .prg files)
- They don't see `tools/vscode-extension/` - that's just for development
